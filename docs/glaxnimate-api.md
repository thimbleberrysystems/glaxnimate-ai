# Glaxnimate Python API — as actually built

Generated by `scripts/introspect_api.py` from the source-built bindings.
**The online docs are out of date** — prefer this file.

## Gotchas (each of these silently wastes an hour)

- `Document` has **no** `.main`. Compositions come from `doc.assets.add_composition()`.
- `render_image(frame)` is on **Composition**, not Document. It returns a `PIL.Image` (RGBA).
- **`Layer.animation.last_frame` defaults to `-1`**, which makes the layer invisible. Set it or every frame renders blank — with no error.
- `io.registry.from_extension(ext, io.Direction.Export)` needs the direction; `save()` takes a **Composition**, not a Document.
- On an IO format, `fmt.name` is a **property** but `fmt.extensions()` is a **method**. Mixing them up is a `TypeError`.
- Rendering at exactly `last_frame` yields an empty image (the range is exclusive at the end).
- **`transform.scale` cannot be written at all** (upstream binding bug). It is an `AnimatedProperty<QVector2D>`, and *every* route silently no-ops: `.value = utils.Vector2D(...)` and `set_keyframe(t, Vector2D)` raise `object of type 'Vector2D' has no len()`, while `Point`/tuple/list assign with no error and leave it at `(1, 1)` with `keyframe_count == 0`. `transform.position` (a `QPointF` property) works fine both ways, so this is specific to the vector channel. Measured consequence: a bouncing ball whose maths asks for `scale.y=0.79` renders 80x80 at every frame — squash-and-stretch never reached the document. Two things do work: a shape's own **`size`** (a `QSizeF` property) animates correctly, and **draw-time scaling** of prop coordinates (`engine/props.py::draw_prop`, which takes `(sx, sy)`).
- **`set_transition` on a scale property segfaults** (upstream binding bug; position and rotation transitions are fine). Write scale keys linear-timed and let the reducer add keys instead — see `engine/bake.py`.
- Entering `environment.Headless()` more than once per process is unstable: documents from an earlier context dangle. Use one process-wide environment (`engine/session.py`).

## Minimal working example

```python
from glaxnimate import model, utils, environment, io

with environment.Headless():
    doc = model.Document("")
    comp = doc.assets.add_composition()
    comp.width, comp.height = 640, 480
    comp.animation.last_frame = 60

    lay = comp.add_shape("Layer")
    lay.animation.first_frame = 0
    lay.animation.last_frame = 60      # REQUIRED: defaults to -1

    lay.add_shape("Fill").color.value = "#e33"   # applies to *following* shapes
    ball = lay.add_shape("Ellipse")
    ball.size.value = utils.Size(80, 80)
    ball.position.set_keyframe(0,  utils.Point(100, 400))
    ball.position.set_keyframe(30, utils.Point(320, 120))

    img = comp.render_image(30)        # -> PIL.Image RGBA
    fmt = io.registry.from_extension("json", io.Direction.Export)
    data = fmt.save(comp)              # -> bytes (Lottie)
```
